Hello world:

import sml.io.println;
println( "Hello world!");

Pierwsza linijka kodu importuje z biblioteki standardowej, z podkatalogu io funkcję println, która w drugiej linijce jest użyta do wyświetlenia napisu "Hello world!".

Sposób zapisu operacji matematycznych:

W większości języków programowania sposób zapisu operacji matematycznych to nie problem i jest oczywisty,
lecz w Montym wygląda to nieco inaczej, ponieważ używana jest odwrotna notacja polska.
To rozwiązanie ma swoje zalety np. brak kolejności wykonywania działań (kolejność można wywnioskować z sposobu zapisu), większa czytelność, szybszy algorytm parsowania.
Zamiast

2 + 2

jest

2 2 +


operator idzie na koniec, a używanie nawiasów w działaniu
(chyba, że używa się do oddzielenia argumentów funkcji od jej nazwy, co jest wymagane) jest niepoprawne i prowadzi do błędu.
To samo dotyczy porównań, przypisań i wszystkich innych operacji:

int a 2 =;
if a 2 == a 3 == |;
a 1 -=;
end;

Na każdy operator przypadają dwa literały, więc NIE można zrobić tak:

2 2 2 +

Zmienne:

int a 1 =;

Jest to deklaracja zmiennej nazywającej się a o typie danych int, który jest statyczny.
Typ statyczny jest takim typem, że po zadeklarowaniu nie można go zmienić.
W języku Monty występują także zmienne o typach dynamicznych. Są to takie zmienne, którym z kolei po zadeklarowaniu można zmienić typ danych na inny podczas działania programu.
Przy deklaracji zmiennej o typie dynamicznym również trzeba go podać, ponieważ dynamiczność typu jest tutaj jedynie możliwością jego zmiany:

dynamic int a 1 =;

Typy danych zmienia się w sposób jawny:

change a to real;

Teraz zmienna a jest typu real i jej wartość wynosi 1.0.
Wszystkich typów danych jest pięć i są nimi:
- int (liczba całkowita),
- real (liczba rzeczywista),
- string (ciąg znaków),
- boolean (prawda albo fałsz),
- any (dowolny typ danych prosty lub instancja struktury).

Pętle:

W języku Monty trzy rodzaje pętli:
- while,
- do while,
- for.

import sml.io.println;
int a 100 =;
while a 100 < a 0 > &;
println(a);
a 1 -=;
end;

import sml.io.println;
int a 100 =;
do while a 100 < a 0 > &;
println(a);
a 1 -=;
end;

Po uruchomieniu obu kodów można zauważyć, że podczas wykonywania pierwszego kodu z pętlą while, nic się nie wyświetliło,
a podczas wykonywania drugiego już tak.
Pętla do while podczas pierwszego powtórzenia nie sprawdza warunku, a pętla while już tak.
Jest jeszcze wspomniania wcześniej pętla for.

import sml.io.println;
for x in "Test";
println(x);
end;

Powyższy kod wyświetli każdą literę napisu "Test" w nowej lini.
Pętle można przerywać słowem kluczowym break; i pomijać aktualne powtórzenie dzięki continue;.

Funkcje:

W Montym występują również funkcje. Poniżej znajduje się przykład funkcji podającej wartość absolutną liczby.

func real abs real x;
if x 0 >=;
return x;
end;
return x!;
end;

Z kolei wywołanie tej funkcji wygląda w sposób pokazany niżej.

abs(-2.3);

Oczywistym faktem jest to, że nic się nie wyświetli po wywołaniu tej funkcji, ponieważ to, że tak się stanie jest napisane dokładnie nigdzie,
aby się tak jednak stało należy podać wywołanie tej funkcji jako argument do funkcji println albo print, którą trzeba najpierw zaimportować z sml.io.

import sml.io.println;
println(abs(-2.3));

Poniżej jest przykład pustych funkcji: bez parametrów i z wieloma parametrami.

func void test1;
end;

func void test2 any a, any b;
end;

Funkcje mogą być zagnieżdżone, a to czy zostaną zadeklarowane przed czy po ich wywołaniu nie ma w Montym znaczenia.

func int fib int n;
return recFib(n, 0, 1);
func int recFib int n, int l, int r;
if n 0 ==;
return l;
end;
return recFib(n 1 -, r, l r +);
end;
end;

Struktury:

Monty jest językiem obiektowym, aczkolwiek szablonem tychże obiektów nie są klasy, a stare dobre struktury.
Struktury mogą w sobie zawierać funkcje, zmienne i inne, zagnieżdżone struktury.
Strukturę inicjalizuje, jeśli istnieje, funkcja void init, przyjmująca dowolną ilość argumentów.
Można porównywać obiekty operatorem ==, jeśli lewy operand zawiera w sobie funkcję boolean equals, przyjmującą jeden argument o typie any, w przeciwnym wypadku wartość zwracana przez operację porównania będzie równa false.
Jeżeli struktura zawiera funkcję string toString nie przyjmującą argumentów, to kiedy zajdzie potrzeba zamiany obiektu na ciąg znaków właśnie ta funkcja zostanie wywołana, w przeciwnym wypadku będzie to nazwa struktury, która musi zaczynać się wielką literą, hash ("#"), numer instancji tej struktury liczony od zera (nazwa "#" numer + +).
Deklaracja struktury oprócz samej struktury deklaruje jeszcze funkcję o jej nazwie, która jeśli struktura zawiera funkcję init, przyjmuje wszystkie jej argumenty i ją wywołuję.
Niezależnie od tego czy funkcja init jest w strukturze, funkcja o nazwie struktury zwraca jej nową instancję.
Jest jeszcze druga funkcja, która jest deklarowana przy deklaracji struktury. Funkcja ta ma nazwę struktury z dodanym jeszcze na początku "is" i sprawdza czy wartość jest instancją tej struktury. Aby obiekt był iterowalny należy w nim umieścić zagnieżdzoną strukturę o nazwie Iterator, w której jest funkcja boolean hasNext i any next, obie funkcje nie mogą przyjmować argumentów.

import sml.io.println;
struct Person;
string firstName;
string lastName;
func void init string firstName, string lastName;
this.firstName firstName =;
this.lastName lastName =;
end;

func boolean equals any other;
return isPerson(other) firstName other.firstName == lastName other.lastName == & &;
end;

func string toString;
return firstName " " lastName + +;
end;
end;
any me1 Person( "Szymon", "Perlicki") =;
any me2 Person( "Szymon", "Perlicki") =;
println(me1);
println(me2);
println(me1 me2 ==); println(me1.firstName);
println(me1.lastName);

Importowanie:

W Montym znajduje się biblioteka standardowa z wieloma modułami.
Można importować całe moduły importując je tak samo jak funkcje lub zmienne, po prostu nie precyzując co chce się z nich zaimportować.
Pierwszym członem przy importowaniu musi być nazwa pliku bez rozszerzenia, folder albo bilbioteka napisana w javie z użyciem specjalnie przygotowanych bibliotek (taką biblioteką jest sml).
Nie można zaimportować całej biblioteki na raz.
Importowanie struktury przebiega tak samo jak importowanie funkcji, ponieważ nie cała struktura jest importowana, a jedynie jej konstruktor.
examples/test.mt

func void t;
end;

Jeśli tym członem jest nazwa pliku bez rozszerzenia to, aby zaimportować wszystkie funckje i zmienne można to tak zostawić, ale jeśli chce się zaimportować konkretną funkcję lub zmienną to po kropce trzeba napisać jej nazwę (jeśli funkcja i zmienna ma taką samą nazwę to obie zostaną zaimportowane).
examples/fromFileImportingTest.mt

import test.t;
t();

Z kolei jeśli na początku jest folder to nie można go całego zaimportować, a należy sprecyzować plik po kropce, tak jak w poprzednim przykładzie po nazwie pliku można napisać co konkretnie chce się zaimportować.
fromFolderImportingTest.mt

import examples.test.t;
t();

W przypadku gdy początkowym członem jest nazwa biblioteki i nie jest nią sml to trzeba wcześniej zadeklarować jej użycie, a potem nazwa modułu (w którym może być inny moduł) albo funkcji lub zmiennej.
Poniżej podam przykład, kiedy z jakiejś przyczyny mamy bilbiotekę sml skompilowaną (pod nazwą sml.jar) jako osobną bibliotekę, a nie bibliotekę wbudowaną w interpreter.

jar sml; # deklaracja użycia biblioteki
import sml.io.println;
println("Test!");

Więcej szczegółów:

W Montym znajduje się jeszcze dużo więcej rzeczy między innymi tablice, listy, stos, funkcje matematyczne, unix time i wiele więcej.
Aby dowiedzieć się więcej na temat języka Monty można przeczytać dokumentację sml (Standard Monty Library).
