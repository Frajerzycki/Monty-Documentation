Hello world:

import sml.io.println;
println( "Hello world");

Pierwsza linijka kodu importuje z biblioteki standardowej, z podkatalogu io funkcję println, która w drugiej linijce jest użyta do wyświetlenia napisu "Hello world".

Sposób zapisu operacji matematycznych:

W większości języków programowania sposób zapisu operacji matematycznych to nie problem i jest oczywisty,
lecz w Montym wygląda to nieco inaczej, ponieważ używana jest odwrotna notacja polska.
To rozwiązanie ma swoje zalety np. brak kolejności wykonywania działań (kolejność można wywnioskować z sposobu zapisu), większa czytelność, szybszy algorytm parsowania.
Zamiast

2 + 2

jest

2 2 +


operator idzie na koniec, a używanie nawiasów w działaniu
(chyba, że używa się do oddzielenia argumentów funkcji od jej nazwy, co jest wymagane) jest niepoprawne i prowadzi do błędu.
To samo dotyczy porównań, przypisań i wszystkich innych operacji:

static int a 2 =;
if a 2 == a 3 == ||;
a 1 -=;
end;

Na każdy operator przypadają dwa literały, więc NIE można zrobić tak:

2 2 2 +

Zmienne:

static int a 1 =;

Jest to deklaracja zmiennej statycznej o nazwie a i typie danych int.
Ale co to właściwie znaczy zmienna statyczna o typie danych int?
Znaczy to, że ta zmienna przechowuje wartość, która jest liczbą całkowitą i po deklaracji nie może zmienić typu danych na inny.
W języku Monty występują również zmienne dynamiczne. Są to takie zmienne, którym po zadeklarowaniu można zmienić typ danych na inny podczas działania programu.
Przy deklaracji dynamicznej zmiennej również trzeba podać typ danych, ponieważ zmienna w tym języku musi mieć zawsze przypisany określony typ danych:

dynamic int a 1 =;

Typy danych zmienia się w sposób jawny:

change a to float;

Teraz zmienna a jest typu float i jej wartość wynosi 1.0.
Wszystkich typów danych jest pięć i są nimi: int (liczba całkowita), float (liczba zmiennoprzecinkowa), string (ciąg znaków), array (tablica mogąca przechowywać dowolne typy danych).

Pętle:

W języku Monty występują dwa rodzaje pętli: while i do while.

import sml.io.println;
static int a 1 =;
while a 2 ==;
println(a);
end;

import sml.io.println;
static int a 1 =;
do while a 2 ==;
println(a);
end;

Po uruchomieniu obu kodów można zauważyć, że podczas wykonywania pierwszego kodu z pętlą while nic się nie wyświetliłą,
a podczas wykonywania drugiego już tak.
Pętla do while podczas pierwszego powtórzenia nie sprawdza warunku, a pętla while już tak.
